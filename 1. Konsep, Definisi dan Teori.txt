Git dan Github merupakan 2 hal yang berbeda
Git = Version Control System. Bisa mengelola perubahan dalam suatu dokumen, program dan sistem informasi lainnya
Version control system : 
   1. sebuah sistem yang menyimpan "rekaman / snapshot" perubahan pada source code
   2. Memungkinkan bekerja berkolaborasi dengan baik
   3. Mengetahui siapa yang melakukan perubahan dan kapan
   4. Memungkinkan untuk kembali ke keadaan sebelum perubahan (checkout)
checkout = berpindah ke suatu commit
Repository / Repo = folder project dalam git
Commit = tahapan yang dilakukan setelah melakukan perubahan untuk memberikan history apa yang telah dilakukan, dan oleh siapa yang akan terlihat pada history
1 commit = 1 snapshot
hash = penanda unik pada suatu commit
Branch digunakan saat bekerja bersama tanpa mengganggu commit master / trial penambahan fitur baru
branch = cabang bebas dari sebuah commit
merge = penggabungan 2 buah branch
Github = layanan cloud untuk menyimpan dan mengelola project / repo git
Push = pengiriman commit ke repo  github
pull = mengunduh commit dari repo github ke git internal komputer
Syarat push dan pull bisa dilakukan : Repo di github harus sebagai remote dengan cara clone
remote = sumber yang memiliki repo
clone = mengambil repo dari remote
